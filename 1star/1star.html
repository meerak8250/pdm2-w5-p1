Instructions

This exercise will walk through how to pass arguments to callback functions, which can be useful when writing reusable event handlers for inputs such as buttons. You will use to to write a single event handler to be used by three different buttons to allow the user to load a different image depending on which button was clicked.

  1. In sketch.js, create a canvas (any size) and three buttons with labels "bear", "cat", and "monkey". Don't worry too much about positioning or styling the buttons, but you can revisit PDM 1 Week 7 lecture 1 if you need a reminder.
  2. Create a global variable called img, which will eventually be used to store an image. In draw(), use the image() function to display img. You won't see anything (except an error message) when you run the code.
  3. Next, handle the error message. The error is occurring because img doesn't have a value yet. Handle this using either approach shown in class (a try-catch or a conditional that only calls image() if img !== undefined).
  4. Define a function called getAnimalImage that takes one parameter called name. Inside the function, add the following line: 
    img = loadImage("assets/" + name + ".jpg");
    This line loads a different image into the img variable based on the value of name. For example, calling getAnimalImage("monkey") will load the monkey image in the assets folder into your sketch.
  5. The final step is to call the getAnimalImage() function when a button is pressed. As a reminder, you can do this by passing the getAnimalImage() function each button's mousePressed() method. So far, you've been told not to use brackets when passing a function as an argument to another function. For example, if one of your buttons is called catButton, you could do this: 
    catButton.mousePressed(getAnimalImage);
    Write code so that each button calls getAnimalImage() as shown above.
  6. If you completed the previous step and tried clicking the buttons, you have probably noticed nothing happens and you get some cryptic errors messages. The problem is that getAnimalImage() expects a name and you haven't provided one. Try modifying the code you wrote in the previous step so that getAnimalImage() is called with an argument for name (it should match an image name in the assets folder) e.g.: 
    catButton.mousePressed(getAnimalImage("cat"));
  7. Now you'll find that an image loads immediately, probably the image associated with the last button created. This is because passing a function call as an argument, rather than just the function name, causes the function to run immediately instead of waiting for the event (mousePressed in this case) to occur. To fix this, you can pass an anonymous function to mousePressed. The anonymous function will then call getAnimalImage() with the appropriate argument. Remember there are two ways to write an anonymous function:
    • Traditional syntax: 
      function() {
    getAnimalImage("name goes here");
}
    • Arrow function syntax: 
      () => getAnimalImage("name goes here")
    Both options are equivalent in this case so choose whichever makes more sense to you. Here is a completed example: 
    catButton.mousePressed(function () {
    getAnimalImage("cat");
})
    Finish the remaining buttons.
